# 🔍 Global Search QA Test Cases

## 📌 Project Overview

This repository contains manual QA test cases designed to validate the **Global Search functionality** of a web application.

The objective of this project is to ensure that the search feature provides accurate, relevant, and efficient results based on user input. It demonstrates my ability to design comprehensive test scenarios and validate critical user-facing functionality.

---

## 🎯 Scope of Testing

The testing focuses on validating the behavior and performance of the Global Search feature across different scenarios.

### Key areas covered:

* Keyword-based search validation
* Exact and partial match results
* Case sensitivity testing
* Handling of special characters
* Empty and invalid input scenarios
* Search result relevance and accuracy
* Performance and response time validation

---

## 🧪 Types of Testing Performed

* Functional Testing
* Negative Testing
* Edge Case Testing
* UI Validation (basic)

## 📝 Test Case Design Approach

Each test case includes:

* Test Case ID
* Module
* Scenario
* Test Steps
* Expected Result
* Priority
* Severity
* Test Type
* Test Data
* Status (Pass/Fail)
* Automation

The test cases are written in a structured and easy-to-understand format to ensure clarity and maintainability.
